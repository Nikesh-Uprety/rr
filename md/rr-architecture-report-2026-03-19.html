Executive summary

This repository is a TypeScript + Node.js monolith: one server process provides a React (Vite) SPA, an Express API (/api/*), and an admin notifications WebSocket (/ws/admin/notifications). The main persistence layer is PostgreSQL accessed through Drizzle ORM.

Technology stack

  • Frontend: React + Vite (client/)
  • Backend: Express (server/)
  • Database: PostgreSQL + Drizzle ORM (shared/schema.ts)
  • Auth: Passport Local + Postgres-backed sessions
  • Realtime: ws WebSocket server
  • Media: local uploads (uploads/*) + Cloudinary
  • Email: Nodemailer (Mailjet via env configuration)

Repository layout (high level)

Entry points and “centers of gravity”

  • Backend entry: server/index.ts
  • API surface: server/routes.ts
  • Auth: server/auth.ts
  • Database: server/db.ts + shared/schema.ts
  • Storage abstraction: server/storage.ts
  • Realtime: server/websocket.ts

Operational dependencies

  • PostgreSQL: required (DATABASE_URL)
  • Cloudinary: media management
  • SMTP (Mailjet): outbound emails
  • Local disk: uploads directory (ensure persistence in hosting)
